CORBA Messaging

Durante muito tempo, o CORBA se baseou em comunicação síncrona, que não é adequada para sistemas mais complexos. Temos problemas como desconexões, dipositivos móveis, para citar poucos. Por isto foi criada esta especificação. Ela define mensagens assíncronas, invocações independentes de tempo e facilidades para especificar qualidade de serviço para as mensagens.

Os seguintes modos de envio de mensagens estão disponíveis:

  • Síncrona: Envia a mensagem e bloqueia, esperando a resposta.
  • Síncrona deferida: Envia a mensagem e continua a execução. A resposta poderá ser pega depois.
  • Oneway: Envia a mensagem. O ORB fará o melhor possível para entregar a mensagem (mas não garante). Não há resposta.
  • Callback: É enviada, junto com a mensagem, uma referência a um objeto que será utilizada para o envio da resposta.
  • Pollig: Mensagem retorna imediatamente um valuetype que pode ser utilizada para esperar a resposta separadamente.
