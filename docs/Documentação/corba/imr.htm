Implementation Repository - IMR

Permite o binding indireto de referência de objetos persistentes. Este fraco acoplamento entre os objetos possibilita a mudança da localização do objeto sem afetar os objetos que o estão acessando (transparência de mobilidade). Sua implementação não é padronizada, extremamente dependente do ORB em questão.

O IMR possui várias responsabilidades: manter um registro de servidores conhecidos, saber que servidor está sendo executado, assim como sua localização, iniciar servidores on demand. Segue abaixo um exemplo de estrutura de dados utilizada pelo IMR para manter tais informações:

Adapter Name Start-up Command Address
Fred rsh bobo "/usr/local/bin/fred-x" bobo:1799
Joe /usr/local/bin/joe
