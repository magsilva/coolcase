General Inter-ORB Protocol (GIOP)

GIOP é uma coleção de mensagens que os ORBs podem utilizar entre si. No entanto, isso não é o bastante. Ele não especifica como essas mensagens serão transportadas. Temos, portanto, uma implementação de GIOP que utiliza TCP/IP para para isso: o Internet Inter-ORB Protocol (IIOP). Isto não é uma opção: todo Object Request Broker deve suportar o IIOP. Para garantir a operação com outras redes, temos o Environment Specific Inter-ORB Protocol (ESIOP), que mapeia as mensagens GIOP para outros protocolos (DCE - Distributed Computing Environment - por exemplo).

Trocando em miudos, nós temos o GIOP estabelecendo as mensagens, o IIOP como padrão de implementação, o ESIOP garantindo a heterogeinidade. Um ORB que tenha um ESIOP não precisa implementar o IIOP: pode simplesmente oferecer uma ponte entre o IIOP e o ESIOP.

O GIOP consiste em três partes:

  • Common Data Representation (CDR): Mapeia tipos da OMG IDL para tipos de baixo nível para uso em redes, além de definir características tais como ordenação de byte variável (entre big e little endian), tipos primitivos alinhados
  • Formato de mensagens GIOP (existem sete)
  • Considerações sobre o transporte de mensagens GIOP: tem de ser orientado a conexão, confiável, ser visto como um fluxo de bytes, notificação de perda de conexão, a conexão inicial pode ser mapeada para uma do modelo TCP/IP.
