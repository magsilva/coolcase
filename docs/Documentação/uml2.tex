A UML é uma linguagem de especificação, visualização, construção e documentação de artefatos de sistemas de software e regras de negócio, assim como outras atividades não relacionadas com software. Até este ponto, ela não tem nada de anormal, várias outras linguagens já foram criadas com objetivos semelhantes: OOSE, OMT, Fuse. O que a destaca é que ela foi criada para ser adequada a uma ampla gama de domínios, tornando-se um padrão para modelagem de sistemas. Isto permite que sejam desenvolvidas ferramentas cada vez melhores, a troca de modelos entre indústrias (uma necessidade real diante da globalização), desenvolvimento de soluç°es em um espaço de tempo muito mais curto, melhores condiç°es para melhorar a qualidade de software, etc.

Para alcançar todos estes objetivos, optou-se por definir a UML numa arquitetura de quatro camadas:

\begin{itemize}
	\item Meta-metamodelo: infraestrutura necessária para criar o metamodelo, a linguagem a ser utilizada para isto.
	\item Metamodelo: instanciação de um ou vários meta-metamodelos, define a linguagem a ser utilizada para definir os modelos.
	\item Modelo: instanciação de um metamodelo, define a linguagem a ser utilizara para definir os objetos do usuário em um dominio especifico.
	\item Objetos do usuário: instanciação de um modelo, define uma informação específica do domínio.
\end{itemize}

 Ele é mais compacto que o metamodelo devido ao seu maior nível de abstração. Seria desejáve que ambos compartilhassem seus projetos, a maneira como são construídos, definidos. Espera-se que, com a UML 2.0, ambos estejam mais próximos um do outro nesses termos.


Um modelo necessita de ser visualizado de diversas maneiras, com diferentes níveis de abstração, para representar de maneira adequada o sistema em questão. Na UML, definem-se os seguinte diagramas para adequar os dados:

\begin{itemize}
	\item Diagrama de caso de uso
	\item Diagrama de classe
	\item Diagramas de comportamento (behavior)
	\begin{itemize}
		\item Diagrama de statechart
		\item Diagrama de atividade
		\item Diagrama de interação
		\begin{itemize}
			\item Diagrama de seqüência
			\item Diagrama de colaboração
		\end{itemize}
	\end{itemize}
	\item Diagramas de implementação
	\begin{itemize}
		\item Diagrama de componentes
		\item Diagrama de deployment
	\end{itemize}
\end{itemize}
